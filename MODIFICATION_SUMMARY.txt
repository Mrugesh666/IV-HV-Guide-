Four core modules have been completely redesigned for seamless 
integration with options_manager.py:

1. ✅ alert_system_v2.py (503 lines)
   - Email & SMS alerts with position context
   - Understands strategy state, breakevens, Greeks
   - Rate-limited to prevent alert spam
   
2. ✅ backtesting_engine_v2.py (476 lines)
   - Replay strategies on historical data
   - Direct integration with options_manager state
   - Calculates metrics: Sharpe, Sortino, Calmar, max drawdown
   
3. ✅ broker_api_handler_v2.py (476 lines)
   - Multi-broker abstraction (Zerodha, Angel, Upstox)
   - Graceful fallback to Yahoo Finance
   - Returns Greeks with option prices
   
4. ✅ performance_analytics_v2.py (521 lines)
   - Trade logging from options_manager state
   - Risk-adjusted metrics (Sharpe, Sortino, etc.)
   - Detailed performance reports & dashboards

5. ✅ complete_integration_example.py (520 lines)
   - Working example showing full workflow
   - Setup → Monitor → Close → Analyze → Backtest
   - Ready to copy & adapt for production

│ 2. BACKTESTING ENGINE v2
├─────────────────────────────────────────────────────────────────┤

🔄 BACKTEST SCENARIO
  ├─ BacktestScenario class
  │  ├─ Single strategy + date range
  │  ├─ Manual trade entry or from options_manager state
  │  ├─ Historical data loading (Yahoo Finance)
  │  └─ Comprehensive metrics calculation
  │
  └─ Methods:
     ✓ add_trade() — manual entry
     ✓ add_trade_from_options_manager_state() — direct import
     ✓ calculate_metrics() — full metrics set
     ✓ get_summary() — text report

📊 METRICS CALCULATED
  ├─ Trade Statistics
  │  ├─ total_trades, winning_trades, losing_trades
  │  ├─ win_rate (%)
  │  └─ consecutive wins/losses
  │
  ├─ Profitability
  │  ├─ total_pnl, avg_pnl, avg_win, avg_loss
  │  ├─ largest_win, largest_loss
  │  └─ roi (%)
  │
  ├─ Risk Metrics
  │  ├─ profit_factor = gross_profit / abs(gross_loss)
  │  ├─ payoff_ratio = avg_win / abs(avg_loss)
  │  ├─ max_drawdown
  │  └─ recovery_factor = total_profit / abs(max_drawdown)
  │
  └─ Risk-Adjusted Returns (Annual)
     ├─ sharpe_ratio
     ├─ sortino_ratio (downside deviation only)
     ├─ calmar_ratio = annual_return / abs(max_drawdown)
     └─ avg_trade_duration

⚙️ BACKTEST ENGINE
  ├─ BacktestEngine class
  │  ├─ Multi-scenario support
  │  ├─ run_all() — execute all scenarios
  │  ├─ compare_strategies() — side-by-side metrics
  │  └─ generate_report() — text report file
  │
  └─ Workflow:
     1. Create scenario(s)
     2. Add trades (manual or from options_manager state)
     3. Run backtest → calculates metrics
     4. Compare strategies → best performer
     5. Generate report → .txt file with full analysis

🔗 OPTIONS MANAGER INTEGRATION
  ├─ Input: Closed position state dict

  │    'net_credit': 250,
  │    'current_pnl': 9375,
  │    ...
  │  }
  │
  └─ Integration points:
     ✓ After position closes: add_trade_from_options_manager_state(state)
     ✓ Monthly/quarterly: backtest accumulated trades
     ✓ Strategy comparison: compare_strategies()
     ✓ Performance tracking: metrics trends

IV. DATA FLOW EXAMPLE: Iron Fly Setup to Close
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PHASE 1: ANALYZE
  ├─ [BrokerAPI] Fetch current spot price
  ├─ [OptionsManager] Determine IV/HV ratio
  └─ Output: Strategy selection (Iron Fly chosen)

PHASE 2: SETUP
  ├─ [OptionsManager] Calculate strikes around ATM
  ├─ [BrokerAPI] Fetch option prices for all legs
  │  └─ Returns: {bid, ask, iv, delta, gamma, theta, vega}
  ├─ [OptionsManager] Calculate entry debit/credit
  ├─ State created with:
  │  {
  │    'strategy': 'Iron Fly',
  │    'entry_spot': 24500,
  │    'upper_be': 24600,
  │    'lower_be': 24350,
  │    'net_credit': 250,
  │    'legs': {...},
  │  }
  └─ [AlertSystem] Send position_opened alert

PHASE 3: LIVE (continuous monitoring loop)
  ├─ Every 1 minute:
  │  ├─ [BrokerAPI] Fetch spot price
  │  ├─ [OptionsManager] Update Greeks, calculate P&L
  │  │
  │  └─ Breach detection:
  │     ├─ IF spot near breakeven
  │     │  └─ [AlertSystem] Send breach alert
  │     │
  │     ├─ IF P&L ≥ 50% of credit
  │     │  ├─ [AlertSystem] Send profit_target alert
  │     │  └─ GOTO PHASE 4
  │     │
  │     └─ IF P&L ≤ -20% of capital
  │        ├─ [AlertSystem] Send stop_loss alert
  │        └─ GOTO PHASE 4

PHASE 4: CLOSED
  ├─ State updated with close details:
  │  {
  │    'phase': 'CLOSED',
  │    'close_reason': 'PROFIT_TARGET',
  │    'exit_spot': 24525,
  │    'current_pnl': 9375,
  │  }
  ├─ [AlertSystem] Send position_closed alert
  ├─ [Analytics] Add trade to log:
  │  log.add_trade_from_options_manager_state(state)
  ├─ [Analytics] Calculate metrics
  └─ [Backtest] Can replay this trade

PHASE 5: ANALYSIS (daily/weekly)
  ├─ [Analytics] generate_summary()
  │  └─ Win rate, Sharpe, max drawdown, etc.
  ├─ [Analytics] generate_detailed_report()
  │  └─ trade_log.json + performance_report.txt
  └─ [Backtest] Compare strategies
     └─ backtest_report.txt

VI. KEY DESIGN PRINCIPLES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ DIRECT STATE INTEGRATION
   └─ All modules accept options_manager state dict directly
      No need for custom wrappers or data transformation

✅ GRACEFUL DEGRADATION
   └─ If broker unavailable → fall back to Yahoo Finance
   └─ If email unavailable → continue with SMS
   └─ If SMS unavailable → continue with email

✅ RATE LIMITING
   └─ Alerts: 5 min (email), 15 min (SMS) cooldown per type
   └─ Prevents alert spam during high volatility

✅ CONTEXT-AWARE
   └─ Alerts include position context (spot, breakevens, Greeks)
   └─ Metrics track strategy-specific performance
   └─ Reports breakdown by strategy

✅ PERSISTENT & AUDITABLE
   └─ All trades logged to JSON (audit trail)
   └─ Performance reports auto-generated
   └─ Backtest results reproducible

✅ MODULAR
   └─ Each module independent (no internal dependencies)
   └─ Can use individually or as complete suite
   └─ Easy to extend or replace components

  ✓ Verify recipient email addresses
  ✓ Check SMTP server/port (gmail: smtp.gmail.com:587)

Problem: Broker connection fails
Solution:
  ✓ Verify API key & access token are valid
  ✓ Check broker is installed (pip install kiteconnect)
  ✓ Falls back to Yahoo Finance automatically
  ✓ Yahoo Finance provides spot price but not options

Problem: Metrics showing empty
Solution:
  ✓ Verify trades are marked as 'CLOSED' status
  ✓ Check trade_log.json file exists
  ✓ Ensure trades have non-zero 'pnl' values
  ✓ Run: python3 -c "from performance_analytics_v2 import TradeLog; print(TradeLog().get_trades())"

  ✓ Ensure position was closed (phase='CLOSED')
  ✓ Check trade_log.json for entries
  ✓ Verify scenario date range includes trades
  ✓ Run: python3 -c "from performance_analytics_v2 import TradeLog; print(len(TradeLog().get_trades()))"
